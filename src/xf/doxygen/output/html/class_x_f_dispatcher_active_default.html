Detailed Description

A dispatcher implementation which may be executed using an internal thread.

The active dispatcher internally requests a Thread from the XFResourceFactory and starts it with the start() method. The thread itself is responsible to execute the protected method execute().

◆ executeOnce()

int XFDispatcherActiveDefault::executeOnce ( )
overridevirtual

Executes once the dispatcher.

Usually, this method must not be called explicitly. The standard way to call the dispatcher is to call execute().

This method can be called by the Thread (or main function) if it must perform concurrent (non-XF related) tasks in addition.

When using this method start() must not be called.
